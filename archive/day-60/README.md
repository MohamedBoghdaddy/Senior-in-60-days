# Day 60 — Capstone Handoff & Interview Prep
![Day](https://img.shields.io/badge/Day-60-blue) ![Area](https://img.shields.io/badge/Area-system-design-lightgrey) ![Priority](https://img.shields.io/badge/Priority-P0-red)

---
### Focus

Finalize the project, tag a release, and prepare for interviews.
### Timebox

~3 hours
### Study

- [Preparing for technical interviews](https://interviewing.io/blog/how-to-prepare-for-your-next-interview)
- [Writing a great portfolio](https://towardsdatascience.com/creating-an-impressive-portfolio-that-gets-you-hired-6a4184e5f09d)
### Build

1. Tag the final release of the project and ensure version numbers are updated.
1. Collect metrics (test coverage, performance numbers) for portfolio presentation.
1. Write an interview cheat sheet summarizing the project, key decisions, and trade-offs.
1. Rehearse a short talk track explaining the project end-to-end (problem, solution, architecture, lessons).
### Assignments

- Commit release tag and update CHANGELOG.
- Add the cheat sheet to `/docs/interview-pack.md`.
### DoD Checklist

- [ ] Release tag created and repository tidy.
- [ ] Interview pack contains succinct explanations and Q&A topics.
- [ ] Personal reflection summarizing the 60-day journey recorded in `notes/day-60.md`.
### Commit Message

`chore: finalize capstone release and prepare interview materials`
### Outcome Artifacts

- Tag or release in version control.
- `docs/interview-pack.md` summarizing project and potential interview topics.
- Reflection note in `/notes/day-60.md`.
### Self‑Review Questions

- What are the proudest achievements of this project?
- How would you improve the project with more time or resources?
