# Core Engineering: LeetCode Strategy

This folder is a senior-focused repository of problem-solving practice.

## Deep-solving method
1. **Brute force**
   - Start with the simplest correct solution.
   - Prove the logic with examples and edge cases.
2. **Optimize**
   - Identify the dominant cost and remove it.
   - Move from O(n^2) to O(n log n) or O(n) where possible.
3. **Complexity analysis**
   - Document time and space costs clearly.
   - Compare best, average, and worst cases.
4. **Tradeoff explanation**
   - Explain why a faster solution is worth the added complexity.
   - Show when slower, simpler code is preferable.
5. **Clean rewrite**
   - Deliver code that is readable, maintainable, and production-ready.

## Folder purpose
- Capture the problems selected by algorithmic pattern.
- Show senior-level evaluation beyond simply solving the prompt.
- Use this space to demonstrate the engineering habit of refactoring with tradeoffs.
