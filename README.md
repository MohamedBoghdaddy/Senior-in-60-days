# Senior-in-60-days

This repository has been transformed from a day-by-day learning scaffold into a senior engineering portfolio organized around real system capabilities.

## What changed

- reorganized from chronological `day-*` folders into a capability-based portfolio under `90-days-fullstack-engineer/`.
- preserved the original daily logs in `archive/`.
- added a production-grade multi-service platform with backend, AI service, frontend, and infrastructure artifacts.
- included senior-level documentation on architecture, tradeoffs, incidents, postmortems, and decision records.

## Key sections

- `90-days-fullstack-engineer/00-overview/` — roadmap, goals, baseline.
- `90-days-fullstack-engineer/01-core-engineering/` — algorithm strategy, SQL, system design.
- `90-days-fullstack-engineer/02-backend-engineering/` — Node and .NET scaffolds, patterns.
- `90-days-fullstack-engineer/03-fullstack-system/` — main platform architecture, backend, frontend, AI service, infra.
- `90-days-fullstack-engineer/04-devops/` — Docker, Kubernetes, CI/CD strategy.
- `90-days-fullstack-engineer/05-production-thinking/` — incident reports, postmortems, scaling.
- `90-days-fullstack-engineer/06-writing-thinking/` — ADRs, design docs, tradeoffs.
- `90-days-fullstack-engineer/07-showcase/` — demo narrative, system walkthrough, interview stories.

## Running the system

The multi-service demo is designed to be run locally with `docker-compose` from `90-days-fullstack-engineer/03-fullstack-system/infra/docker-compose.yml`.

## Historical archive

- The original `day-01` to `day-60` folders are now preserved under `archive/`.
- Those folders remain as historical evidence but are not the primary portfolio narrative.

## Senior signal

This repository now emphasizes:

- system thinking and architecture decisions
- production readiness and failure handling
- real service design and tradeoff documentation
- measurable progression from learning scaffold to engineering portfolio
