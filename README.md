# Senior Engineer in 90 Days

A structured, evidence-driven engineering transformation system.
Not a course collection. Not motivational content. A daily operating system for becoming stronger.

---

## Honest audit of where this repo started

Before the 90-day plan can be trusted, it needs to be honest.

Previous state of this repo:
- Had 60 archive days, but the notes folder contained 60 empty files. No actual learning was captured.
- Had a resource table mapping 60 days to React, Next.js, Supabase, React Native — misaligned with the actual goal.
- Had good structural scaffolding but zero evidence of execution.
- Had system design docs, ADR templates, incident reports — all placeholders, not earned artifacts.

The problem was not the structure. The problem was no daily execution discipline.

This rewrite fixes that.

---

## The actual goal

Move from junior/mid execution ability toward senior engineering thinking across:

- Node.js (internals, production patterns, architecture)
- .NET (clean architecture, SOLID, testing)
- LeetCode (Python + C++, pattern-based, tracked)
- SQL and databases (schema design, indexing, transactions, performance)
- Docker and Kubernetes (production containers, orchestration, observability)
- Backend architecture and system design (ADRs, design docs, tradeoffs)
- Production thinking (incidents, debugging, reliability, CI/CD)
- Engineering communication (design docs, code reviews, STAR stories)

---

## What makes this system different from a course list

Every day has:
- A specific objective (not "learn X")
- A specific study resource (not a whole course)
- A specific build task (something you commit to the repo)
- A reflection output (notes, writeup, or artifact)
- A reason why this day matters

No day ends without evidence in the repo.

---

## 90-Day structure

| Phase | Days | Focus |
|-------|------|-------|
| Phase 1: Foundation Upgrade | 1–30 | Node.js internals, SQL depth, LeetCode arrays/trees/graphs in Python, backend architecture basics |
| Phase 2: Backend & Architecture Depth | 31–60 | .NET clean architecture, system design drills, code review thinking, debugging, production Node.js |
| Phase 3: Production & Scalability | 61–90 | Docker, Kubernetes, observability, CI/CD, scaling patterns, senior communication, interview prep |

## The single rule

Every day you must commit at least one file with actual content.
Notes count. LeetCode solutions count. Code count. Writeups count. Nothing else does.

An empty notes file is the same as not showing up.

```bash
docker-compose up
```
