# Day 59 — Portfolio Packaging: Deploy & Demo
![Day](https://img.shields.io/badge/Day-59-blue) ![Area](https://img.shields.io/badge/Area-docs-lightgrey) ![Priority](https://img.shields.io/badge/Priority-P0-red)

---
### Focus

Deploy the application and ensure all components are accessible for demonstration.
### Timebox

~3 hours
### Study

- [Vercel deployment docs](https://vercel.com/docs)
- [Railway deployment guide](https://docs.railway.app/deploy/)
### Build

1. Deploy the web front-end to a platform like Vercel with environment variables configured.
1. Deploy the API to a cloud service (e.g., Railway, Fly.io) with a separate database instance.
1. Deploy the mobile app as a preview build (Expo EAS) and generate installation links.
1. Test the deployed environments thoroughly to ensure features work as expected.
### Assignments

- Commit deployment configurations and update documentation with deployment URLs.
- Ensure environment variables are secure and documented.
### DoD Checklist

- [ ] Web, API, and mobile deployments are publicly accessible.
- [ ] Features work end-to-end in the deployed environments.
- [ ] Deployment instructions documented for future updates.
### Commit Message

`deploy: deploy front-end, API, and mobile app; update docs with links`
### Outcome Artifacts

- Deployment configuration files.
- Updated docs with live URLs and deployment notes.
### Self‑Review Questions

- Are there any environment differences that could cause issues?
- Is the deployment process repeatable and documented?
